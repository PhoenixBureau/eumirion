O Page Actions

A "page action" is a bit of text in a page's "text" text that refers to a Page URL and matches a certain pattern, namely up to thirty-two a-z letters, a colon ":" character and then a Page URL "/nnnnnnnn/nnnnnnnn"

action:/nnnnnnnn/nnnnnnnn

text:

Render the text of the target page into this page as if it was part of this page's text.

Be careful not to write a page action that points to the page it's on or you'll get an infinite loop which will cause the server to return a 500 Server Error response!

div:

Like text: but it puts the target page's text into a <DIV> element.

p:

Like text: but it puts the target page's text into a <P> element.

ol:

Like text: but it puts the target page's text into a <OL> element.

Use this with the li: page action to build Ordered Lists.

ul:

Like text: but it puts the target page's text into a <UL> element.

Use this with the li: page action to build Unordered Lists.

li:

Like text: but it puts the target page's text into a <LI> (List Item) element.

Use this with the ol: and ul: page actions to build Ordered and Unordered Lists.

section:

Just like text: but it also puts the target page's title into an <H2> element immediately preceding the text, giving the effect of a subtitled section in the page.

link:

This renders a hyperlink to the target page using the page's title as the link text. This action is the default, and you can leave off the "link:" part entirely and just write the Page URL to get the same effect.

door:

Make a hyperlink like link: but also add a CSS class to the link for special rendering. This is something of an experiment, for making "doors" to other pages.

css:

Render the text of the target page as a CSS <style>, this is the way to add styling to pages (along with the cssclass: page action.)

cssclass:

This page action adds the text of the target page as a "class" attribute to the enclosing HTML element into which it is rendered.

This way you can add styling to e.g. individual paragraphs and whatnot.

joy:

This is the only way to add "joy" text to Page Data. (Commands can affect a page's "joy" expression of course. That's what they are for. This is the way to bootstrap the "joy" expressions of command pages from the Built-in Joy Functions, Combinators, and Definitions.)

It constructs "joy" expressions on pages from the text of (the same or other) pages. It scans the text of the target page up until the first cssclass: page action (so that the CSS class for Joy code can also serve as the delimiter.) That text is converted to a joy expression and concat'd to the page's existing "joy" expression.

One thing to be aware of here is that multiple renderings (either from posting the edit form or from clicking command: buttons) will cause any joy: actions to be run and more stuff concat'd to the page's existing "joy" expression. Typically you'll use this page action to build command expressions and then delete it so that it doesn't mess them up inadvertently later.

command:

This page action renders a button that, when clicked, will cause the target page's "joy" expression to be evaluated with this page's "joy" expression as the stack. If the evaluation succeeds this page's "joy" expression is replaced by the resultant stack. In any event, the page is re-rendered.

The text of the button is the title of the target page.

stack:

Render target page's "joy" expression into the page as an Ordered List. This is mostly a diagnostic tool to let you see a page's expression in a direct and unambiguous way, although it can certainly be used creatively as a content rendering tool.

stackinto:

Render target page's "joy" expression into the page as text. Especially, any Page Actions appearing in any text in the expression will take effect.

This is primary way let a page's "joy" expression affect the rendering of the content. Any command: buttons on a page will first modify its "joy" expression (here used as the stack) and then re-render the page. You can see and example of this on the See Animation in Action page.
