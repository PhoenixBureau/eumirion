O joy:

This is the only way to add "joy" text to Page Data. (Commands can affect a page's "joy" expression of course. That's what they are for. This is the way to bootstrap the "joy" expressions of command pages from the Built-in Joy Functions, Combinators, and Definitions.)

It constructs "joy" expressions on pages from the text of (the same or other) pages. It scans the text of the target page up until the first cssclass: page action (so that the CSS class for Joy code can also serve as the delimiter.) That text is converted to a joy expression and concat'd to the page's existing "joy" expression.

One thing to be aware of here is that multiple renderings (either from posting the edit form or from clicking command: buttons) will cause any joy: actions to be run and more stuff concat'd to the page's existing "joy" expression. Typically you'll use this page action to build command expressions and then delete it so that it doesn't mess them up inadvertently later.
